## 概要

**ComfyUI-Autocomplete-Plus** は、[ComfyUI](https://github.com/comfyanonymous/ComfyUI) の任意のテキストエリアに複数の入力支援機能を提供するカスタムノードです。現在はDanbooruとe621のタグに対応しています（e621は一部の機能が未対応です）。

## このフォークについて

このリポジトリは [newtextdoc1111/ComfyUI-Autocomplete-Plus](https://github.com/newtextdoc1111/ComfyUI-Autocomplete-Plus) の継続メンテナンス版です。上流プロジェクトは長期間更新されておらず、最近の ComfyUI フロントエンド変更への対応も進んでいません。このフォークは、現在の ComfyUI で拡張機能を引き続き利用できるようにし、互換性修正を継続するために作成されました。

### 上流版との主な違い

#### 互換性とメンテナンス

- 現行の ComfyUI フロントエンドに合わせて継続的にメンテナンス
- **Nodes 2.0** のテキスト入力と、**サブグラフノード**に昇格された入力の両方に対応し、元の内部ノードとウィジェットまで解決

#### ローカル優先のデータソース

- 同梱の Hugging Face CSV を即座に表示し、LoRA Manager と Danbooru はバックグラウンドで並行して補足
- 長い Danbooru クエリは部分一致を使い、作品名の断片からキャラクタータグも検索可能
- LoRA Manager のタグ、LoRA、Embedding、Wildcard API を再利用し、LoRA Manager 自身の入力欄では重複補完を自動回避
- 補完と関連タグの末尾に専用の取得元列を設け、最終的に採用された最優先の取得元を `CSV`、`LM`、`API` のうち 1 つだけ表示。重複バッジによる圧迫を防ぎ、ホバーで詳細を確認可能
- Danbooru、e621、CSV、LoRA Manager の候補を取得元で分割せず、人気度基準でまとめて並べ替え

#### より自然な補完フロー

- 入力途中では補完を再表示し、確定後は関連タグへ直接切り替え、末尾のカンマからも直前のタグを認識
- Danbooru を待たずにローカルの完全な関連タグを表示。API 固有行は後から追加し、順位、選択、スクロール位置を変更しません
- 挿入時に周囲のカンマと改行を再利用・正規化し、空タグや区切りの重複を防止
- 関連タグ挿入後も探索を継続でき、パネル固定と `Ctrl+Shift+Space` による表示に対応
- 既存タグをグレー表示し、再選択時は重複挿入せず元の位置へ移動
- 行とパネルから Wiki を直接開け、キーボード選択中は `F1` でも表示可能

#### 高速で安定した大規模リスト

- 安全上限付きの完全な結果スナップショットを最初に生成し、スクロールバーの最終長を即座に確定
- 固定行高の仮想リストは可視行と少数のバッファだけを描画し、スクロールから検索、ページ取得、翻訳を発生させません
- フレーム単位の検索統合、候補上限、CSV の分割解析、追記専用翻訳インデックス、SQLite 処理の遅延、関連タグ計算のキャッシュで入力応答を維持
- 交互の行背景、常に見える選択アクセント、安定したキーボード移動、スクロールバー領域の予約、画面に応じた配置を採用
- 表示中のパネル幅を固定し、仮想スクロールや遅れて届く翻訳による左右の揺れを防止
- 英語タグへより広い領域を割り当て、翻訳列は短い固定幅に調整。省略された値はホバーで全文表示
- 狭いウィンドウでは列幅、余白、補助操作を自動的に圧縮。補完はキャレット付近に配置し、関連タグは入力欄の空きが広い側を優先し、左右に収まらない場合は上または下へ切り替え

#### 永続オンラインサービスと翻訳

- Danbooru の補完と関連タグを永続キャッシュし、バックグラウンド更新、オフラインフォールバック、同時リクエストの統合、上限付き LRU 整理、統計、手動消去に対応
- 上位結果を先に翻訳し、その後も未翻訳行を制御されたバッチで順次補完。完了したバッチは直ちに一覧へストリーム反映し、処理中の行だけにアニメーション付き 3 点インジケーターを表示。原文のまま、または対象言語の文字を含まない疑似翻訳は拒否し、アーティストタグは翻訳せず原名を表示
- 実際の選択リストによるモデル検出、稼働確認、保存済み API Key のオンデマンド表示、任意の思考強度に対応
- 設定の先頭に**オンラインサービス**を配置し、Danbooru と翻訳を個別に切り替え可能。無効化しても既存キャッシュは保持
- オンラインサービス画面を再設計し、コンパクトな機能スイッチ、確認しやすい稼働状態、控えめなキャッシュ操作、DeepSeek に集中したフォーム、API Key の表示切り替え、要約付き詳細設定、明確な保存操作に加え、ボーダーレスの階層と柔らかなエッジシャドウを採用
- オンライン補足はローカル入力をブロックせず、障害時も入力を妨げずに設定画面へ診断情報を保持

#### 積み重ねた細かな工夫

- 別名プレビューを現在の ComfyUI 表示言語で絞り込み、完全な別名セットは引き続き検索可能
- 一般、アーティスト、作品、キャラクター、メタ、モデルなどにカテゴリ別 Emoji とローカライズ済みの説明を追加
- すべての設定名とツールチップに加え、見出し、読み込み・空状態、操作ボタン、Wiki リンク、取得元バッジを英語・簡体字中国語・繁体字中国語・日本語に対応
- 英語・簡体字中国語・日本語のドキュメントを同期し、大型機能だけでなく操作上の工夫や細かな改善も記載

元のプロジェクトは引き続きこのフォークの基盤です。既存の機能とクレジットは可能な限り維持します。

## インストール

### ComfyUI-Manager

1. [ComfyUI-Manager](https://github.com/Comfy-Org/ComfyUI-Manager) で `Autocomplete-Plus` と検索して表示されたカスタムノードをインストールし、再起動します
2. 起動時に必要な CSV データが HuggingFace から自動的にダウンロードされます

### マニュアル

1. このリポジトリを ComfyUI の `custom_nodes` フォルダにクローンまたはコピーします  
    `git clone https://github.com/Aaalice233/ComfyUI-Autocomplete-Aaalice.git`
2. ComfyUI を起動します。起動時に必要な CSV データが HuggingFace から自動的にダウンロードされます

## オートコンプリート

テキスト入力エリアで文字を入力すると、テキストに部分一致するタグを投稿数の多い順で表示します。上下キーで選択、EnterかTabキーで選択したタグを挿入できます。

- タグのエイリアスも検索対象に含まれます。日本語のひらがな、カタカナは区別せず検索されます
- タグのカテゴリ毎に色分けされます。色分けのルールは Danbooru と同じです
- 候補ごとにカテゴリ別 Emoji を表示し、カーソルを合わせるとカテゴリとデータソースを確認できます
- 一致した全ソースの候補は、画面に表示される投稿数（人気度）の降順で並べます。人気度が同じ場合だけ、一致精度とソース優先度を比較します
- 現在選択中の候補は、一覧表示直後の先頭行やマウスホバー中でもアクセント付きのハイライトを維持します
- 入力済みのタグはグレーアウトで表示されます
- Danbooruとe621のタグを同時に表示出来ます。設定から優先順位を変更できます
- LoraとEmbeddingの入力補完に対応しています。設定から有効・無効を切り替えられます
- 「📖」アイコンをクリックするとタグのWikiページを開きます。キーボードで選択中の場合は `F1` キーで開くことが出来ます

## 関連タグ

![ss02](https://github.com/user-attachments/assets/854571cd-01eb-4e92-a118-2303bec0b175)

テキスト入力エリアの任意のタグを選択すると、関連性の高いタグを一覧表示します。タグをクリックするか、キーボードの上下キーで選択後にEnterまたはTabキーでタグを挿入出来ます。UIは編集中のテキストエリアを基準に位置とサイズが自動で調整されます。

- ローカル CSV の完全な共起スナップショットを先に即時表示し、その後で Danbooru 公式関連タグ API をバックグラウンド取得します。API 固有タグだけを末尾へ追加するため、ローカル順位や現在の選択を動かさず、パネル表示も待機しません
- 入力途中のタグをクリックすると、オートコンプリートが再表示されます。クリックしたタグに共起データがある場合のみ関連タグを表示するため、空の関連タグパネルが有効な補完候補を隠すことはありません
- Enter、Tab、またはマウスクリックで補完候補を確定すると、完成したタグに共起データがある場合は関連タグがすぐに表示されます
- タグ末尾のカンマ、またはカンマ直後のスペースをクリックすると、直前のタグに対する関連タグが表示されます
- 関連タグ挿入時は周辺の区切りを統一し、プロンプト末尾、既存タグの間、改行直前でも重複カンマを作りません
- 関連タグを挿入すると、パネルはそのタグの共起候補へすぐ切り替わり、連続して選択できます。固定中のパネルは固定したタグを維持します
- クリックでオートコンプリートと関連タグを切り替える場合、一方を開くともう一方は閉じます（関連タグパネルを固定している場合を除きます）

- 表示位置は、テキストエリアの下部を基本とし、空きスペースに応じて上下に自動調整されます
  - ヘッダーの「↕️|↔️」ボタンで上下と左右の表示位置に切り替えられます
- ヘッダーの「📌|🎯」ボタンで表示する関連タグの固定状態を切り替えられます。固定状態で閉じたい場合はEscキーを押します
- ヘッダーのタグをクリックするとタグのWikiページを開きます
- 入力済みのタグはグレーアウトで表示されます。グレーアウトしたタグを挿入しようとした場合、代わりに入力済みのタグを選択状態にします
- `Ctrl+Shift+Space` キーでカーソル位置の関連タグを表示できます

## 自動フォーマット

テキスト入力エリアがフォーカスを失った際（例：外側をクリック、Tabキーを押す）、プロンプトテキストを自動的にフォーマットする機能です。大量のテキストを編集する際に可読性を向上させる事が出来ます。

詳細な動作は以下になります：
- 各タグの後に自動的にカンマとスペースを追加し、適切に区切ります
- タグ間の余分なカンマとスペースを削除します
- キーボードショートカット `Alt+Shift+F` を使って手動でフォーマットすることも出来ます（キーバインドはComfyUIの設定から変更可能です）
- 設定から機能の有効化/無効化を切り替えられます

> [!NOTE]
> 一部のノードはエラーに繋がるため、自動フォーマットは実行されません。  
> 例： [Power Puter (rgthree)](https://github.com/rgthree/rgthree-comfy/wiki/Node:-Power-Puter) の `code` フィールド、 [LoraLoaderBlockWeight (Inspire)](https://github.com/ltdrdata/ComfyUI-Inspire-Pack) の `block_vector` フィールド

## CSV データ

動作には基本となる CSV データが2つ必要です。これらは [HuggingFace](https://huggingface.co/datasets/newtextdoc1111/danbooru-tag-csv) で管理されており、 ComfyUI にインストール後の初回起動時に自動でダウンロードされるのでセットアップは不要です。
基本 CSV ファイルはHuggingFaceで公開されているDanbooruデータセットを元にしているので、Danbooruサイトの投稿数や関連タグの情報と異なる場合があります。

> [!IMPORTANT]
> 基本 CSV にはSFW, NSFW両方のタグが含まれています。

**danbooru_tags.csv**

タグ名、カテゴリ、投稿数、エイリアス（日本語、中国語、韓国語を含む）の情報を持つオートコンプリート用のタグ情報 CSV ファイルです。このカラム構成は [DominikDoom/a1111-sd-webui-tagcomplete](https://github.com/DominikDoom/a1111-sd-webui-tagcomplete) で使用されているものと同じです。

タグ情報は以下の条件でフィルタリングされています。
- 投稿数100件以上
- 投稿画像のスコアが5以上
- カテゴリが `general, character, copyright` のいずれか
- タグ名に `(cosplay)` が含まれていない

**danbooru_tags_cooccurrence.**

タグペアとその共起回数を記録した、関連タグ計算用の CSV ファイルです。

タグペアはタグ情報 CSV から、さらに以下の条件でフィルタリングされています。
- 共起回数が100件以上

### e621 CSV

現在、e621用 CSV の自動ダウンロードは未対応なため `danbooru_tags.csv` と同じ構造の CSV を `e621_tags.csv` という名前でデータフォルダーに手動配置してください。  
また、関連タグ表示も同様に未対応です。

### ユーザーCSV

ユーザーが自身で用意した CSV を使用することも可能です。 CSV ファイルは以下の命名規則に従って `data` フォルダーに配置してください。

- **オートコンプリート用 CSV**: <danbooru | e621>_tags*.csv
- **関連タグ用 CSV**: <danbooru | e621>_tags_cooccurrence*.csv

例として、よく使うメタタグを `danbooru_tags_meta.csv` の名前で `data` フォルダーに配置することでオートコンプリート候補に追加できます。
ヘッダー行はなくても構いません。反映にはブラウザのリロードが必要です。

**メタタグの例**
```csv
tag,category,count,alias
masterpiece,5,9999999,
best_quality,5,9999999,
high_quality,5,9999999,
normal_quality,5,9999999,
low_quality,5,9999999,
worst_quality,5,9999999,
```

ブラウザリロード時、ロードされる CSV ファイル一覧をComfyUIのコンソールのログで確認出来ます。ログ出力に含まれていない場合はファイル名が命名規則に沿っているか確認してください。

**ComfyUIコンソールログ出力の例:**
```
[Autocomplete-Plus] CSV file status:
  * Danbooru -> base: True, extra: danbooru_tags_meta.csv // ここに表示されていればメタタグを入力補完できます
  * E621     -> base: False, extra: 
```

>[!NOTE]
> ユーザー CSV が複数ある場合アルファベット順に読み込まれます。同じタグが複数のファイルに存在する場合は先に読み込まれた方が保持されます。基本 CSV は最後にロードされます。

### 複数タグの一括挿入機能（疑似 Chants）

`""` （ダブルクォーテーション）で複数タグを囲むことで、よく使うタグを一括で挿入できます。
これは [DominikDoom/a1111-sd-webui-tagcomplete](https://github.com/DominikDoom/a1111-sd-webui-tagcomplete?tab=readme-ov-file#chants) で **Chants** と呼ばれる機能に似ています。

例として以下のCSVを用意することで、`<c:Basic-HighQuality>` や `<c:Basic-Negative>` と入力して対応したタグを素早く挿入出来ます。

**`danbooru_tags_chants.csv`:**
```
"masterpiece, best quality, high quality, highres, ultra-detailed",5,9999999,<c:Basic-HighQuality>
"(worst quality, low quality:1.4), normal quality",5,9999999,<c:Basic-Negative>
```

>[!TIP]
> * `""` で囲まれたテキストは `()` （括弧）がエスケープされません。元々括弧が含まれるタグはエスケープした状態でCSVに記述してください。 例： `copyright_(series)` -> `copyright_\(series\)`
> * エイリアス列も `""` に対応しているので、複数のエイリアスを付けられます

## 設定

### タグソース

> [!NOTE]
> Danbooruやe621等のタグデータの提供元を「タグソース」と呼びます

- **Autocomplete Tag Source**: オートコンプリート候補に表示するタグソース。「all」を選択するとロード済みの全てのタグソースを表示します
- **Primary source for 'all' Source**: `Autocomplete Tag Source` が「all」のとき、ここで指定したタグソースが優先して表示されます
- **タグカテゴリのアイコン位置**: カテゴリ Emoji の表示位置を指定します。カーソルを合わせるとカテゴリとデータソースが表示され、「hidden」で非表示にできます

### オートコンプリート

- **Enable Autocomplete**: オートコンプリート機能の有効化/無効化
- **補完結果の最大数**: 完全なメモリ内スナップショットの安全上限（既定値：1000、最大：2000）。DOM には可視行だけを配置します
- **Auto-Insert Comma**: タグの挿入時、末尾にカンマを追加する
- **Replace '_' with 'Space'**: タグ挿入時にアンダースコアをスペースに置き換えます。この設定は関連タグ表示にも影響します
- **String to add before artist tags**: アーティストタグを挿入する際に付加する文字列。 Anima モデルの場合は「@」を指定します
- **Enable Loras and Embeddings**: LoraとEmbeddingを候補に表示する
- **Use Fast Search**: インデックス検索を使用します（既定で有効）。50,000 件以上のタグを読み込んだソースは、以前の保存設定が無効でも UI を止める逐次走査を回避します

### 関連タグ

- **Enable Related Tags**: 関連タグ機能の有効化/無効化
- **共起タグ結果の最大数**: 完全な共起スナップショットの安全上限（既定値：25000）。DOM には可視行だけを配置します
- **Default Display Position**: ComfyUI起動時のデフォルト表示位置
- **Related Tags Trigger Mode** : 入力済みのタグの関連タグを表示する際、どの操作をトリガーとするか（クリックのみ、Ctrl+クリック）

### 表示

- **Hide Alias**: オートコンプリートと関連タグで表示されるエイリアス列の非表示/表示を切り替え（デフォルトは表示です）

### 自動フォーマット

- **Enable Auto Format**: テキストエリアがフォーカスを失った際にプロンプトテキストを自動的にフォーマットする機能の有効化/無効化
- **Auto Format Trigger**: フォーマットを適用するタイミングを選択します
  - **自動**: テキスト欄からフォーカスが外れた際に自動でフォーマットします
  - **手動**: キーボードショートカットでのみフォーマットします（デフォルト: `Alt+Shift+F`）
- **行末にカンマを使用**: 有効時、フォーマット時にすべての行末がカンマで終わるようになります。\n無効にすると行末のカンマが削除されます
- **先頭・末尾の空白を削除**: 有効時、プロンプトの先頭と末尾にある空行やスペースをすべて削除します

### LoRA Manager 連携

[ComfyUI LoRA Manager](https://github.com/willmiao/ComfyUI-Lora-Manager) がインストールされている場合、既定の「自動」モードはローカルの `/api/lm/custom-words/search`、`/api/lm/loras/relative-paths`、`/api/lm/embeddings/relative-paths`、`/api/lm/wildcards/search` インデックスから候補を補完します。API が利用できない場合は入力を妨げず、標準データへ戻ります。LoRA Manager 自身の補完入力欄は自動的に除外され、他のサードパーティ製ノードは「除外するノードタイプ」に追加できます。

### オンライン補完と翻訳

オートコンプリートは安全上限付きの完全なローカル結果スナップショットを即座に計算し、LoRA Manager と最大 200 件の Danbooru スナップショットを並行取得して統合します。有効文字が 2～3 文字の間は低コストな前方一致を使い、4 文字以上では部分一致へ切り替えるため、`wuthering_wave` から `_(wuthering_waves)` で終わるキャラクタータグも取得できます。新しい検索方式は別のキャッシュバージョンを使い、旧前方一致結果を再利用しません。スクロール中はスナップショットを固定し、スクロール処理からデータソースへアクセスしません。投稿数 0 の空タグ、非推奨タグ、未対応カテゴリ、旧翻訳辞書に残った投稿数 0 のオンラインタグは除外されます。タグ列と翻訳列には個別の最大幅があり、省略された全文はホバーで確認できます。

関連タグも同じローカル優先方式です。ローカル CSV の完全なスナップショットを先に表示し、その後で Danbooru 公式 `/related_tag.json` から Jaccard 順の最大 500 件を一度だけバックグラウンド取得します。API 固有行だけを追加するため、遅延結果は既存順位やスクロール位置を変えず、スクロール自体からリクエストも発生しません。

成功した Danbooru 補完ページと関連タグスナップショットは `completion_cache.sqlite3` にまとめて保存され、ブラウザー更新や ComfyUI 再起動後も利用できますが、第二の永続タグ DB にはなりません。空でない結果は 7 日間新鮮とみなし、期限後は古い結果を即座に返しながらバックグラウンド更新します。Danbooru が利用できない場合は最大 90 日の古い結果へフォールバックし、空結果はより短い期間だけ保持します。同一クエリの同時未命中は上流リクエストを 1 回にまとめ、補完のページ継続判定にはフィルター後の件数ではなく Danbooru の生ページ件数を使います。LRU クリーンアップは最大 5000 キャッシュエントリを保持し、オンラインサービス画面から件数・容量の確認と翻訳 DB に影響しない消去ができます。

補完と関連タグの各行には低コントラストの取得元チップを 1 つだけ表示します。同梱データセットは `CSV`、LoRA Manager は `LM`、Danbooru は `API` です。同じ候補を複数ソースが返した場合は、統合結果で実際に採用された最優先の取得元だけを表示します。ホバーすると完全な取得元名を確認できます。

**Autocomplete Plus → オンラインサービス → オンライン補完と翻訳** で DeepSeek を設定します。クエリが安定すると補完の先頭 200 件と関連タグの先頭 300 件を単一ジョブの前方へ置き、その後ろへ現在の言語の訳がない全項目を追加します。フロントエンドは完全なジョブを一度だけ送信し、バッチ分割と同時実行はバックエンドが担当します。設定した `concurrency` は全ジョブで共有する DeepSeek のグローバル上限として適用され、同じタグは SingleFlight で統合されます。これによりブラウザー側の固定 3 並行という隠れた上限をなくし、`batch_size` と `concurrency` の両方を実際の処理へ反映します。既定のバッチサイズは 20 です。翻訳処理はスクロールから完全に分離され、バックエンドで完了したバッチは最も遅いバッチやスクロールによる再描画を待たず、現在の仮想ウィンドウへ直ちにストリーム反映されます。未完了の行だけが翻訳列にアニメーション付き 3 点インジケーターを表示し、キャッシュ済み訳は即座に表示されます。原 Tag と同一、または対象言語の文字体系を含まない結果は拒否され、有効な CSV 訳を上書きしません。永続ストアを開く際に既存の無効訳も削除します。アーティストタグは DeepSeek へ送信せず、翻訳列に原 Tag を直接表示します。LoRA、Embedding、Wildcard も翻訳対象外です。成功した翻訳は `translations.sqlite3` に保存され、一致するすべての booru 検索インデックスにも追加されます。

設定画面には「Danbooru API 補足を有効化」と「自動翻訳を有効化」の独立スイッチがあります。Danbooru を無効にすると補完と関連タグの API リクエストを停止し、永続キャッシュや翻訳辞書にある API 専用候補も一覧から除外します。翻訳を無効にすると辞書の読み込みと自動翻訳リクエストを停止しますが、翻訳 DB は削除せず再有効化後に再利用できます。オンラインソースの確認、利用可能な全モデルを明確な選択リストへ取得、選択モデルの稼働確認、保存済み API Key のオンデマンド表示、Danbooru 結果キャッシュの統計・消去、翻訳の詳細設定も同じ画面で操作できます。

入力補完と関連タグ画面では現在の ComfyUI 表示言語に合う別名だけを表示しますが、完全な別名は検索できます。設定と新しい翻訳辞書は ComfyUI ユーザーディレクトリの `autocomplete-plus/` に保存され、旧ライブタグ DB と CSV は読み込まず、自動削除もしません。

## 上級者向け設定

### 起動時のCSV更新チェックを無効化する

デフォルトの動作では、ComfyUI起動時に一定の間隔で CSV ファイルの更新チェックとダウンロード行います。
インターネットにアクセス出来ない環境で起動した場合、タイムアウトが発生するまで起動が遅延する事があります。

以下の手順を行う事により、ComfyUI起動時のチェック処理をスキップする事が出来ます。

1. このカスタムノードをインストールした状態でComfyUIを一度起動し、 `csv_meta.json` ファイルを生成する  
  `csv_meta.json` はこのカスタムノードのフォルダー直下に作成されます
2. `csv_meat.json` をテキストエディターで開き、`check_updates_on_startup` の値を `true` -> `false` に変更し保存する  
  `check_updates_on_startup` が存在しない場合、 `version` の下に追記してください

**変更後の `csv_meta.json`：**
```json
{
  "version": 1,
  "check_updates_on_startup": false,
  ...
}
```

**補足事項：**
- `check_updates_on_startup` の値を再び `true` にするか、 `version` が切り替わるまでチェック処理は行われなくなります
- `check_updates_on_startup` が `false` でも、Autocompelte Plusの設定から `Check CSV updates` のボタンを押す事で手動チェックが可能です

## 動作に関する詳細

## オートコンプリートの高速検索について

高速インデックス検索は既定で有効です。ローカル候補を即座に表示し、LoRA Manager、Danbooru、翻訳結果は後から段階的に追加されます。50,000 件以上のタグを読み込んだソースは、古い無効設定が残っていても逐次走査を自動的に回避します。

キーボードイベント内で同期検索を実行せず、入力を次のフレームにまとめて処理します。各データソースは上限付きの候補だけを収集してから順位付けし、CSV 解析は小さなチャンクごとにメインスレッドを譲ります。実行時翻訳は独立した追記専用エイリアスインデックスへ保存するため、入力中に大規模な FlexSearch 基本インデックスを再構築しません。共起データは UI に必要な高頻度ペアだけをメモリに保持し、結果もキャッシュします。

- 読み込む CSV ファイルに大量のタグやエイリアスが含まれている場合。タグの合計が **10万件** を超える場合は特に有用です
- プロンプト入力でタグのカンマ区切りの代わりに自然言語を使う場合

**ブラウザ起動時の動作**

高速検索にはタグインデックスの構築が必要です。大規模ソースの構築中は一時的にそのソースをスキップし、データ全体の同期走査で候補の表示や非表示を止めないようにします。
`v1.3.0` 時点では構築が完了したタイミングはブラウザーの開発ツールにのみが表示されます。これは将来のバージョンで改善予定です。

例として、約22万件のタグのインデックス構築が完了した際は以下のようなログが記録されます。
```
[Autocomplete-Plus] Building 221787 index for danbooru took 9398.70ms.
```

> [!NOTE]
> - インデックス構築は設定から高速検索を無効にしてもバックグラウンドで行われます
> - 高速検索の実現に全文検索ライブラリの [nextapps-de/flexsearch](https://github.com/nextapps-de/flexsearch) を使用しています 

## 既知の問題

### パフォーマンス

- CSV ファイルの容量が大きいため、ブラウザの起動時間が長くなる場合があります
- ブラウザ上で高速に動作させるためにメモリを消費します。ComfyUIが動作するスペックのマシンであれば問題にはならないと思います
